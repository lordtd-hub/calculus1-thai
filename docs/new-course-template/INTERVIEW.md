# 🎤 INTERVIEW — คำถามที่ต้องถามอาจารย์

> Claude Code: ถามทีละข้อ เก็บคำตอบลงตัวแปร (mental note)
> ห้ามเดาคำตอบเอง ถ้าอาจารย์บอกให้ข้ามก็เก็บเป็น "skip"
> หลังครบทุกข้อ สรุปเป็น Markdown list ให้ยืนยัน

---

## Section A — ข้อมูลวิชา (บังคับ)

**A1. ชื่อวิชา (ภาษาไทย)**
> ตัวอย่าง: "พีชคณิตเชิงเส้น", "แคลคูลัส 2", "สถิติเบื้องต้น"

**A2. ชื่อวิชา (ภาษาอังกฤษ)**
> ตัวอย่าง: "Linear Algebra", "Calculus II", "Introductory Statistics"
> ใช้สร้าง `course_id` แบบ snake_case (e.g. `linalg_thai`, `calc2_thai`, `stats_thai`)
> — ถ้าอาจารย์ยังไม่คิดชื่อ repo ช่วยเสนอให้

**A3. รหัสวิชา (ถ้ามี)**
> ตัวอย่าง: "SCMA101", "SCMA203"
> ไม่บังคับ — ใช้ประกอบข้อมูลแสดงใน index.html

**A4. ชื่ออาจารย์ผู้ดูแล (attribution)**
> ตัวอย่าง: "อาจารย์ ดร. สมชาย ใจดี"
> ใส่ใน index.html ตรงส่วน "แนวคิดโดย..."

---

## Section B — CLOs (Course Learning Outcomes)

**B1. จำนวน CLO**
> ตัวอย่าง: 4, 5, หรือ 6
> ถ้าอาจารย์ไม่แน่ใจ — ขอดู course syllabus หรือ มคอ.3

**B2. ขอ CLO ทั้งหมด พิมพ์ทีละบรรทัดได้เลย**
> ตัวอย่าง format:
> ```
> CLO1: อธิบายและคำนวณเวกเตอร์
> CLO2: แก้ระบบสมการเชิงเส้น
> CLO3: คำนวณและตีความเมทริกซ์
> CLO4: ประยุกต์การแปลงเชิงเส้น
> ```

---

## Section C — Topics (หัวข้อหลัก)

**C1. หัวข้อหลักในคอร์ส (3–6 หัวข้อ)**
> ขอเรียงตามลำดับการสอน
> ตัวอย่าง (Linear Algebra):
> 1. เวกเตอร์ (vectors)
> 2. เมทริกซ์ (matrices)
> 3. ระบบสมการเชิงเส้น (linear systems)
> 4. ดีเทอร์มิแนนต์ (determinants)
> 5. ค่าเฉพาะและเวกเตอร์เฉพาะ (eigenvalues/eigenvectors)

**C2. สำหรับแต่ละหัวข้อ map ไป CLO ไหน**
> ตัวอย่าง: "เวกเตอร์ → CLO1", "เมทริกซ์ → CLO3"

**C3. อิโมจิ/ไอคอนของแต่ละหัวข้อ (optional)**
> ถ้าอาจารย์ไม่เลือก Claude จะเสนอให้
> ตัวอย่าง: vectors → ➡️, matrices → ▦, eigen → 🌀

## Section D — Pages (หน้าเว็บที่อยากให้มี)

**D1. หน้าแนะนำ (intro.html) — เก็บหรือลบ?**
> หน้า intro ใน calc1 เป็นการลากจุดเข้าใกล้ลิมิต
> วิชาใหม่อาจจะไม่ต้องมี — ถามว่าเก็บไหม (เก็บ = อาจารย์จะคิดการโต้ตอบเองทีหลัง)

**D2. เกม guess / match — เก็บหรือลบ?**
> เก็บได้ทั้งคู่ / เก็บอย่างใดอย่างหนึ่ง / ลบทั้งคู่
> ถ้าเก็บ อาจารย์จะต้องเขียน question bank ใหม่ภายหลัง

**D3. หน้า "แอปประยุกต์" (diff-app / integrate-app) — เก็บหรือลบ?**
> ใน calc1 คือหน้ารวม interactive applications
> วิชาใหม่อาจอยากมี 1–2 หน้าแบบนี้ (เช่น "Linear Algebra apps" โชว์การแปลงเชิงเส้น)
> ถาม: ต้องการไหม — ถ้าต้องการจะใช้ชื่อหน้าอะไร

**D4. หน้าภารกิจ (missions / SBRA) — เก็บไว้ไหม?**
> แนะนำให้เก็บเสมอ (หัวใจของ framework) แต่ถามอาจารย์เพื่อให้แน่ใจ

## Section E — Content language / style

**E1. ภาษาหลัก**
> ไทย (default) / อังกฤษ / ผสม
> ถ้าไม่ใช่ไทย → Claude Code ต้อง find-replace Thai strings หลายจุด

**E2. วิชานี้ใช้สูตรคณิต (KaTeX) ไหม?**
> ใช่ → เก็บ KaTeX + math.js CDN
> ไม่ → ลบออก ไฟล์จะเบาลง

**E3. วิชานี้ต้องวาดกราฟ/รูปเรขาคณิต (canvas) ไหม?**
> ใช่ → เก็บ `js/graph.js`
> ไม่ → ลบ (ประหยัด ~16KB)

## Section F — Visual / Deploy (optional)

**F1. สีหลักของธีม**
> default = ม่วง + ชมพู (`#8b5cf6` + `#fb7185`)
> อยากเปลี่ยนไหม? ถ้าใช่ ขอ color code (primary + accent1)

**F2. Repo URL (ถ้ามี)**
> ถ้าอาจารย์สร้าง GitHub repo ไว้แล้ว — ขอ URL เพื่อ push ให้ตอน Phase 4
> ถ้ายัง — Claude Code แค่ init local, อาจารย์ push เองทีหลัง

**F3. Badge theme (optional)**
> ใน calc1 มี 7 badges (starter → grandmaster)
> อยากเก็บจำนวนเท่าเดิม หรือปรับ?
> อยาก rename badges ไหม (e.g. "นักพีชคณิต" แทน "นักหาอนุพันธ์")

## Tips สำหรับ Claude Code

- **อย่าถามทุกข้อในคำถามเดียว** — ถาม 2-3 ข้อต่อรอบ อาจารย์จะตอบได้ไม่พลาด
- ถ้าอาจารย์ข้ามข้อ — เก็บเป็น `default` (ดู MIGRATION_GUIDE.md หา default values)
- ถ้าอาจารย์ตอบแบบไม่ชัดเจน — เสนอ 2-3 ทางเลือกแล้วให้เลือก
- **ห้าม** ใส่ "สมมุติว่า..." ในสรุป ต้องเป็นคำตอบจริงจากอาจารย์
